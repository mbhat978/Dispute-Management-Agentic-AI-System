# Bug Fixes - April 16, 2026

## Issues Identified and Resolved

### Issue 1: SSE Connection Error - Missing ticket_id in Agent Updates

**Problem:**
After submitting a dispute request, the agent activity was not appearing in the UI live feed. The SSE events were being broadcast with `ticket_id=None`, causing the frontend to fail to display the updates properly.

**Root Cause:**
In `backend/main.py`, the `broadcast_stream_event` calls were using `final_state.get("ticket_id")` which returned `None` because the ticket_id was not being properly propagated through the LangGraph state. The ticket was created in the database with a valid ID (e.g., 25), but this ID was not accessible from the final_state dictionary.

**Solution:**
Modified two locations in `backend/main.py`:

1. **Line 1069** - Processing completed event:
   - Changed from: `"ticket_id": final_state.get("ticket_id")`
   - Changed to: `"ticket_id": new_ticket.id`

2. **Line 1081** - Response payload:
   - Changed from: `"ticket_id": final_state.get("ticket_id")`
   - Changed to: `"ticket_id": new_ticket.id`
   - Also changed: `"customer_id": final_state.get("customer_id")`
   - To: `"customer_id": request.customer_id`

**Impact:**
- SSE events now correctly include the ticket_id
- Frontend can properly display agent activities in the live feed
- All broadcast events are properly associated with the correct dispute ticket

### Issue 2: Backend Auto-Restart During Dispute Processing

**Problem:**
While processing a dispute, the backend application was shutting down automatically and restarting. The logs showed:
```
WARNING: WatchFiles detected changes in 'mcp_servers\core_banking_server.py'. Reloading...
INFO: Shutting down
```

This caused:
- Interruption of the dispute processing workflow
- SSE connection drops
- Poor user experience with connection errors in the frontend

**Root Cause:**
Uvicorn's `--reload` flag with WatchFiles was monitoring all Python files in the backend directory, including the `mcp_servers/` directory. When the MCP (Model Context Protocol) tools were being called during the investigation phase, the file system activity in the `mcp_servers/` directory triggered the auto-reload mechanism.

**Solution:**
Modified `run_app.bat` to exclude the `mcp_servers/` directory from the reload watch:

- **Line 49** - Backend startup command:
  - Changed from: `uvicorn main:app --reload --port 8000`
  - Changed to: `uvicorn main:app --reload --port 8000 --reload-exclude mcp_servers/*`

**Impact:**
- Backend no longer restarts during dispute processing
- MCP tool calls don't trigger file watch events
- Continuous SSE connections maintained throughout the workflow
- Improved stability and user experience
